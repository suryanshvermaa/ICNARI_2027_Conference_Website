# Backend Routes (Frontend Reference)

This folder documents the **HTTP routes** exposed by the Drogon backend.

- Base URL (local): `http://localhost:3000` (or `PORT` from `.env`)
- Versioned API prefix: `/api/v1`
- Health route has no `/api/v1` prefix: `GET /health`

The canonical contract is `openapi.yaml` (in this same folder). This README is a quick summary for frontend usage.

## Response envelope

All endpoints return a JSON envelope:

### Success

```json
{
  "success": true,
  "message": "...",
  "data": {}
}
```

`data` is omitted when an endpoint does not return anything.

### Error

```json
{
  "success": false,
  "message": "..."
}
```

## Authentication (JWT)

Protected routes use Drogon's `AuthFilter`.

- Header: `Authorization: Bearer <jwt>`
- Missing/invalid token: `401` with `{"success":false,"message":"Unauthorized: ..."}`

JWT contents (as used by the backend):

- `userId` (integer)
- `role` (string)

Implementation detail: `AuthFilter` injects `userId` and `role` into request parameters for downstream handlers.

## Auth endpoints

Base: `/api/v1/auth`

- `POST /api/v1/auth/login` (local)
  - Body: `{ "email": string, "password": string }`
- `POST /api/v1/auth/signup` (local)
  - Body: `{ "name": string, "email": string, "password": string }`
- `POST /api/v1/auth/firebase/login`
  - Body: `{ "token": "<firebase_id_token>" }`
- `POST /api/v1/auth/firebase/signup`
  - Body: `{ "token": "<firebase_id_token>" }`

## Users endpoints

Base: `/api/v1/users`

- `GET /api/v1/users` (admin only)
  - Query: `page` (default `1`), `limit` (default `10`)
- `GET /api/v1/users/profile`
  - Query: `id` (required)
- `PUT /api/v1/users` (authenticated)
  - Body: any of `{ "name", "email", "password", "role", "profilePicture" }`
- `DELETE /api/v1/users/{id}` (admin only)
- `POST /api/v1/users/uploadProfilePicture` (authenticated)
  - Body: `{ "file_extension": "png" }`
- `POST /api/v1/users/makeAdmin` (admin only)
  - No body (promotes the *current JWT user* to admin as implemented)

## Pagination

### Users
- Query: `page` (default `1`), `limit` (default `10`)
- Constraint: `limit` must be `1..19` (else `400`)

### Notifications
- Query: `page` (default `1`), `limit` (default `10`), `type`
- `type` allowed: `HIGHLIGHTS` | `RECENT_UPDATES`

## Profile picture upload flow (pre-signed URL)

1) Call `POST /api/v1/users/uploadProfilePicture` with JSON `{ "file_extension": "png" }` and Bearer token.
2) Response contains `data.upload_url`.
3) Frontend uploads the file bytes to the `upload_url` using **HTTP PUT**.

Notes:
- The backend does not currently validate MIME type/content-type.
- The pre-signed URL is generated by AWS SDK and is time-limited.

## Gallery endpoints

Base: `/api/v1/gallery`

- `GET /api/v1/gallery/images`
  - Query: `page` (default `1`), `limit` (default `10`)
- `GET /api/v1/gallery/image/{id}`
- `POST /api/v1/gallery/image/signed-url` (admin only)
  - Body: `{ "title": "some_title" }`
  - Response: `data.signedUrl` (HTTP PUT pre-signed URL)
- `POST /api/v1/gallery/image` (admin only)
  - Body: `{ "title": "...", "tags": "tag1,tag2", "imageObjectKey": "<s3_key_or_url>" }`
- `DELETE /api/v1/gallery/image/{id}` (admin only)
